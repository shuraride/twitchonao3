How to use it

For this section, parts of the HTML will underline in pink on hover, and parts of the CSS will underline in orange on hover.

Adding Users and User Colours

Everything in this section goes in between the <div class="chat-box"></div> tags. If it's not inside those tags, it won't appear properly.

To add new users, simply paste the following into your HTML:

<p><b>Username:<b> Text text text</p>

This turns out as:

I'm A Random Colour: Text text text

Using this code, the usernames will infinitely cycle through 7 different colours.

But what if you want a person to have a specific colour?

In the CSS, you'll find this code:

#workskin .chat-box .mod {
font-weight: bold;
color: green;
}

The ".mod" here is a special classifier that will override the randomized colour. (The reason it's called "mod" is because the green is the colour Twitch uses for its moderators.) You can use it like so:

<p><span class="mod">I'm Green:</span> Text text text</p>

This will result in:

I'm Green: Text text text

Of course, you can change the colour in #workskin .chat-box .mod to anything you want. Here's another example using Nightbot, a very common helper bot on Twitch that typically uses a purple username:

#workskin .chat-box .nightbot {
font-weight: bold;
color: #8A2BE2;
}

and the HTML:

<p><span class="nightbot">I'm Purple:</span> Text text text</p>

This results in:

I'm Purple: Text text text

The mod and nightbot colours are included in the default code because they're two of the most common specific colours I personally am using with this skin. But you can just copy and paste either of the #workskin .chat-box .WORD blocks and change it to any word you want, then use that word in the HTML.

Note that if you make a new unique username class and want the underline that automatically appears when you hover over the username, you will have to add your new .chat-box .WORD to the following part of the code:

#workskin .chat-box b:hover,
#workskin .chat-box .mod:hover,
#workskin .chat-box .nightbot:hover {
border-radius: 5px;
cursor: pointer;
text-decoration: underline;
}

Adding Replies

To add replies, put this HTML above the message that is responding [to the other message]:

<p><span class="reply">| In reply to @USERNAME: TEXT TE...</span><br />

In total, it will look something like:

<p><span class="reply">| In reply to @USERNAME: TEXT TE...</span><br />
<b>Username:<b> Text text text</p>

Adding Images

To add badges (the little icons beside the usernames), use:

<img src="imageurl" class="mod" alt="alttext">

The mod class here will automatically resize it to be 18x18 px.

To add emotes in the messages, just use the normal image code:

<img src="imageurl" alt="alttext">

The emotes will be automatically resize to 28x28 px.

Using Them All Together

Let's say you want to have a message with badges, a special username colour, a response, and an emote all in one - an exact example being Nightbot's message in the above chat box.

The code would be something like:

<p><span class="reply">| In reply to @AuraFavourite: PLS SING INST...</span><br />
<img src="imageurl" class="mod" alt="alttext"><p><span class="nightbot">Nightbot:</span> You have been timed out for 5 minutes. Please refrain from typing the same message in capslock. <img src="imageurl" alt="alttext"></p>

Tips

If you are going to be using a lot of CSS/HTML in your fic, I highly recommend building it in VS Code instead of on AO3. In VS Code, you can deploy a live version of your file that will update as you work, rather than you having to go back/forth between tabs on AO3. It'll also flag any incorrect elements. A guide on how to link your CSS and HTML in VS Code can be found here, and here is the live server extension.

Of course, writing your HTML in VS Code isn't required, and you can still do it on AO3. But using VS Code will save you a lot of trouble and headaches; while it may seem intimidating at first if you've never done any programming, once you get used to it, you'll prefer it. Trust me!
